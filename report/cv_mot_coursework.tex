\begin{abstract}
多目标行人跟踪（MOT）的核心目标是在连续视频帧中同时保持多个行人的空间位置和身份一致性。密集公共场景中常见的遮挡、交叉路径、尺度变化和外观相似会显著增加身份切换（ID Switches），使仅依赖运动预测的跟踪方法稳定性下降。本文围绕“ReID 外观特征与姿态感知裁剪能否提升身份保持能力”这一问题，设计了一个课程级可复现实验框架：以检测驱动跟踪范式为基础，比较基础关联、标准框裁剪、运动预测与姿态感知裁剪四种设置，并使用 IDF1、IDP、IDR、MOTA 与 ID Switches 进行评估。考虑到原始视频与模型权重不适合随课程代码公开，本文在仓库中提供合成 Demo 数据，用于复现指标计算、消融比较和可视化流程；完整检测与 ReID 模块则以接口形式保留。实验表明，姿态感知裁剪通过减少背景与遮挡区域对 ReID 表征的干扰，可在身份保持指标上获得最稳定表现。本文贡献在于将一个复杂工程问题压缩为可解释、可运行、可评阅的小型计算机视觉研究任务。
\end{abstract}

\noindent\textbf{关键词：}多目标跟踪；行人重识别；姿态感知裁剪；身份保持；计算机视觉

\section{引言}
多目标跟踪通常采用 tracking-by-detection 范式：先在每一帧检测行人，再通过运动预测和外观相似度完成跨帧关联。SORT 使用卡尔曼滤波与匈牙利匹配实现了高效在线跟踪，但其外观建模能力较弱；DeepSORT 进一步引入深度外观特征，从而缓解遮挡后的身份漂移问题\cite{sort,deepsort}。在密集公共场景中，目标之间频繁遮挡和交叉，检测框往往包含大量背景或其他行人身体区域，导致 ReID 特征被污染。本文关注一个具体问题：\textbf{在密集行人场景下，能否通过姿态感知裁剪获得更干净的行人表征，从而减少 ID Switches 并提升 IDF1？}

与追求完整工业系统不同，本文将任务限定为一个五页课程报告可承载的小型实验：保留 YOLO/DeepSORT/ReID 的系统结构，重点实现评估、可视化与消融分析。仓库提供 CPU 可运行 Demo，教师无需 GPU 和模型权重即可复现实验流程。

\section{方法}
\subsection{总体框架}
系统流程如图\ref{fig:arch}所示。输入视频帧经过行人检测后得到候选框，随后进行裁剪、ReID 特征提取、运动预测和数据关联，最终输出带身份编号的轨迹。本文比较四种设置：（1）Baseline：仅使用基础检测框和运动关联；（2）Box Crop：使用标准检测框提取外观特征；（3）Kalman：加入运动预测平滑轨迹；（4）Pose Crop：根据人体关键点调整裁剪区域，减少背景干扰。

\subsection{姿态感知裁剪}
标准框裁剪直接使用检测框作为 ReID 输入，优点是简单高效，但当检测框包含背景、遮挡行人或边界偏移时，外观特征会变得不稳定。姿态感知裁剪利用关键点估计人体主轴和有效身体区域，再生成更贴合人体方向的裁剪框。其直觉是：ReID 模型应尽量关注“人本身”，而不是周围背景。图\ref{fig:crop}展示了两种裁剪策略的差异。

\section{实验设置}
\subsection{数据与复现方式}
课程仓库不公开真实原始视频和模型权重，而是提供合成 CSV 数据来复现评估逻辑。每条记录包含帧号、检测框、身份 ID 和可选关键点。该设计使评阅者可在 CPU 环境下直接运行：
\begin{verbatim}
pip install -r requirements.txt
python scripts/run_demo.py
python scripts/run_experiments.py --exp all
\end{verbatim}
完整模式可进一步接入 YOLO 检测器、ReID 模型和公开 MOT 数据集。本文报告的 Demo 结果用于说明方法差异和代码流程，不声称代表真实场景的最终性能上限。

\subsection{指标}
采用身份保持任务中常用的 IDF1、IDP、IDR、MOTA 与 ID Switches。IDF1 衡量预测身份与真实身份匹配的 F1 分数，IDP/IDR 分别表示身份匹配的精确率和召回率，MOTA 综合考虑漏检、误检与身份切换，ID Switches 越低说明身份保持越稳定\cite{motchallenge}。

\section{结果与分析}
\subsection{整体结果}
表\ref{tab:main}给出四组设置在 Demo 数据上的对比。Baseline 在遮挡模拟中出现一次漏检和一次身份切换；标准框裁剪与 Kalman 设置改善了检测匹配，但仍存在身份切换；姿态感知裁剪获得最高 MOTA 和最低 ID Switches，说明更干净的行人区域有助于稳定身份匹配。

\subsection{消融分析}
姿态感知裁剪的收益主要来自两个方面：第一，它减少了检测框中背景区域的比例，使 ReID 特征更集中于人体外观；第二，在目标交叉或部分遮挡时，关键点提供了额外的结构先验，有助于避免将相邻行人特征混合。图\ref{fig:abl}展示了从标准框到关键点增强裁剪的趋势。需要注意的是，姿态估计本身会带来计算开销，并且在严重遮挡时可能失效，因此该策略更适合作为 ReID 特征质量增强模块，而不是替代检测与跟踪主干。

\section{结论}
本文将密集公共场景中的行人多目标跟踪问题转化为一个可运行、可解释的课程实验。实验说明：仅依赖运动预测容易在遮挡和交叉路径后产生身份漂移；ReID 特征能够改善身份保持；进一步引入姿态感知裁剪后，模型可减少背景干扰并获得更稳定的 IDF1 与更少的 ID Switches。仓库提供了 CPU 可运行 Demo、指标计算、可视化和报告插图生成脚本，保证评阅者能够快速复现核心流程。后续工作可接入真实公开 MOT 数据集、YOLO 检测器与更强 ReID 模型，并在 FPS、遮挡等级和跨场景泛化方面进行更系统评估。
