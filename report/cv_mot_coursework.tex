\begin{abstract}
多目标行人跟踪（MOT）的关键不仅是检测行人位置，还要在遮挡、交叉和检测抖动时维持身份编号一致。本文实现了一个面向密集公共场景的运动--外观双分支跟踪系统：运动分支使用 Kalman 滤波预测短时位置，外观分支使用 ReID 特征描述行人身份，二者融合后由匈牙利算法完成全局一对一匹配。实验采用双层设计：一层对齐 MOT17/MOT20 的标准格式与评测协议，另一层构建 Wuzhou\_MidRoad 自标注真实验证集（2474 帧、117 个 ID、33846 个标注框）。系统从 Baseline、Kalman、ReID、姿态裁剪到 BIoU/低置信度扩展逐步迭代；最终在真实场景上取得 IDF1 73.26\%、MOTA 55.49\%、ID Switches 236。报告同时给出 ReID 训练日志曲线、消融结果和误差分析，以体现课程项目的算法设计、数据构建和复现完整性。
\end{abstract}
\vspace{-0.4em}
\noindent\textbf{关键词：}多目标跟踪；ReID；Kalman 滤波；匈牙利匹配；BIoU；自标注验证集

\section{问题定义与系统思路}
密集公共场景中的行人跟踪难点是“持续确认同一个人”。若仅依赖单帧 IoU 或最近邻匹配，目标交叉后容易 ID Switch；若只使用外观特征，检测框包含背景或遮挡时 ReID 特征又会被污染。因此本文将任务拆成两个互补分支：运动分支负责短时连续性，外观分支负责身份一致性。整体流程为：检测框输入后，Kalman 滤波预测已有轨迹下一帧位置；ReID 网络提取检测框外观特征；再将运动代价与外观代价加权融合，最后由匈牙利算法求解全局最小代价匹配。

Kalman 状态向量设为
\begin{equation}
\mathbf{x}=[c_x,c_y,a,h,\dot c_x,\dot c_y,\dot a,\dot h]^T,
\end{equation}
其中 $(c_x,c_y)$ 为框中心，$a$ 为宽高比，$h$ 为高度，后四维为速度。关联总代价为
\begin{equation}
C=\lambda C_{motion}+(1-\lambda)C_{app}.
\end{equation}
$C_{motion}$ 约束短时位移，$C_{app}$ 约束外观身份。相比局部贪心匹配，匈牙利算法能保证每帧多轨迹、多检测框之间的一对一全局分配，避免一个检测框被多条轨迹竞争。

\section{低置信度目标与 BIoU 关联}
真实视频中，远距离行人或遮挡行人常产生低置信、小尺寸、偏移的检测框。直接丢弃会降低 IDR 和 MOTA；直接创建新轨迹又会引入误检。因此本文采用“低置信度补充匹配”策略：低置信目标不直接确认新 ID，而只允许与已有轨迹匹配。IoU 严格比较预测框与检测框交并比；BIoU（Buffered IoU）在关联阶段临时扩大预测框或候选框的搜索区域，适合检测框偏小、遮挡导致边界不准的情况。该扩展不是修改标注框，而是在匹配阶段提高弱目标召回，并继续通过门控阈值限制误匹配。

\section{双层实验数据与迭代设置}
本文不只在合成 Demo 上展示效果，而是采用双层实验设计。第一层使用 MOT17/MOT20 的公开数据格式和评价协议，证明系统遵循标准 MOT 任务定义；第二层构建 Wuzhou\_MidRoad 真实场景验证集。该视频约 99 秒、2475 帧，本文将人工标注/核验后的 \texttt{gt.txt} 整理为 MOT 格式，其中包含 2474 帧、117 个行人身份和 33846 个标注框，并与跟踪器输出 \texttt{Wuzhou\_MidRoad.txt} 逐帧匹配评估。

为体现课程项目的进步迭代，实验设置为五个版本：V0 只使用 IoU 关联；V1 加入 Kalman 运动预测；V2 加入 ReID 外观分支；V3 使用姿态感知裁剪减少背景干扰；V4 引入 BIoU 与低置信度目标搜索扩展，提高真实遮挡场景中的召回。

\section{ReID 训练与评价指标}
外观分支使用 OSNet-AIN / MobileNet Hybrid 作为 ReID 特征提取模型，在 Market1501 与 MSMT17 数据上训练。训练采用 SGD 优化器和 Cosine Annealing 学习率调度，batch size 为 32，输入尺寸为 $256\times128$。训练日志 \texttt{original\_project/run\_logs/reid\_training\_log.txt} 记录 178 个 epoch，最佳 checkpoint 出现在 epoch 178，验证 mAP 为 0.6272。图\ref{fig:train} 由日志直接解析生成，展示 loss 下降、学习率衰减和验证 mAP 提升。

评价指标采用 MOT 常用定义。IDP 表示身份匹配精确率，IDR 表示身份匹配召回率，IDF1 是二者调和平均；MOTA 综合考虑漏检、误检和身份切换；ID Switches 统计同一 GT 身份被分配到不同预测 ID 的次数。评价脚本在每帧内按 IoU 阈值和匈牙利算法匹配 GT 与预测框，累计 TP、FP、FN 和身份切换。

\section{实验结果与误差分析}
Demo 消融结果见表\ref{tab:demo}。Baseline 已有较高 IDF1，但仍有身份切换；加入 Box Crop/ReID 后身份匹配更稳定；Pose Crop 进一步减少背景干扰，使 ID Switches 降为 0。Kalman 单独使用可平滑轨迹，但缺少外观约束时仍可能在交叉场景中混淆身份，因此运动分支与外观分支需要联合使用。

\section{实现细节、复现性与总结}
项目代码围绕可复现评阅组织：\texttt{src/evaluation/metric.py} 实现 MOT 风格指标统计；\texttt{src/tracking/tracker.py} 实现 Kalman 状态预测、轨迹更新与外观特征平滑；\texttt{src/cropping/crop\_strategy.py} 实现标准框裁剪和姿态感知裁剪；\texttt{scripts/run\_experiments.py} 复现 Demo 消融；\texttt{scripts/run\_original\_project.py} 复现 Wuzhou 真实评估；\texttt{scripts/generate\_training\_curve.py} 从日志重绘训练曲线；\texttt{scripts/generate\_enhanced\_figures.py} 可从视频自动抽帧生成真实背景图。

综上，本文完成了一个从算法设计、训练日志、标准评测到自标注真实验证均可复现的 MOT 课程项目。方法上解释了 Kalman、匈牙利、ReID、姿态裁剪、BIoU 与低置信度扩展的作用；实验上展示了 Baseline 到优化版本的迭代过程，并用 Wuzhou\_MidRoad 揭示真实遮挡条件下的主要误差。后续可继续导出真实场景 V0--V4 中间 tracker 文件，形成更严格的真实消融表，并补充 Rank-1/CMC 等 ReID 评价。
