\noindent\textbf{摘 要：}多目标行人跟踪（MOT）的关键不仅是检测行人位置，还要在遮挡、交叉、尺度变化和检测抖动时维持身份编号一致。本文实现了一个面向密集公共场景的运动--外观双分支跟踪系统：运动分支使用 Kalman 滤波预测短时位置，外观分支使用 ReID 特征描述行人身份，二者融合后由匈牙利算法完成全局一对一匹配。实验采用双层设计：一层对齐 MOT17/MOT20 的标准格式与评测协议，另一层构建 Wuzhou\_MidRoad 自标注真实验证集（2474 帧、117 个 ID、33846 个标注框）。系统从基线关联、Kalman 预测、ReID 表征、姿态裁剪到 BIoU/低置信度扩展逐步迭代；最终在真实场景上取得 IDF1 73.26\%、MOTA 55.49\%、ID Switches 236。本文同时给出训练过程摘要、消融结果、路线分析和误差解释，以呈现系统设计、数据构建与实验验证的完整流程。

\noindent\textbf{关键词：}多目标跟踪，行人重识别，卡尔曼滤波，匈牙利匹配，Buffered IoU

\section{研究背景与任务定义}
密集公共场景中的行人跟踪难点在于跨帧维持稳定身份。检测器只能给出单帧中的行人位置，而 MOT 系统还需要判断当前帧目标与历史轨迹之间的身份对应关系。若仅依赖单帧 IoU 或最近邻匹配，目标交叉后容易发生身份切换；若只使用外观特征，检测框中的背景、遮挡物或低分辨率行人又会污染 ReID 表征。因此本文将任务拆分为两个互补分支：运动分支建模短时位置连续性，外观分支约束跨帧身份一致性。

从相关研究看，SORT[1] 证明了 Kalman 滤波与匈牙利匹配可以构成简洁高效的在线跟踪基线，但其主要依赖运动与框重叠，在遮挡和交叉场景中容易发生身份切换；DeepSORT[2] 引入深度外观度量，缓解了只看运动造成的身份混淆；ByteTrack[4] 进一步指出低置信检测框不应被简单丢弃，因为遮挡目标常以低分形式出现；BoT-SORT[5] 和 StrongSORT[6] 则从相机运动补偿、ReID 特征和轨迹管理角度强化关联鲁棒性；OC-SORT[7] 与 Deep OC-SORT[8] 强调观测中心的状态修正和自适应 ReID 使用。因此，本文并不声称提出新的最优跟踪算法，而是在现有强基线思想上构建可复现的工程化跟踪流程：将运动预测、外观度量、低分框补偿、边界框缓冲和自标注验证拆分为可分析模块，并说明各模块对应的失败现象。这样的组织方式避免简单堆叠已有算法，也便于通过消融实验分析改进来源。

系统流程如下：输入检测框后，Kalman 滤波首先预测已有轨迹在当前帧中的先验位置；ReID 网络随后提取候选检测框的外观特征；系统再将运动代价与外观代价加权融合，并由匈牙利算法求解全局最小代价匹配。

Kalman 状态向量设为
\begin{equation}
\mathbf{x}=[c_x,c_y,a,h,\dot c_x,\dot c_y,\dot a,\dot h]^T,
\end{equation}
其中 $(c_x,c_y)$ 为框中心，$a$ 为宽高比，$h$ 为高度，后四维为速度。预测阶段根据上一帧状态给出当前帧的先验框，更新阶段再用成功匹配的检测框修正状态。关联总代价为
\begin{equation}
C=\lambda C_{motion}+(1-\lambda)C_{app},
\end{equation}
其中 $C_{motion}$ 由 IoU/门控距离刻画，$C_{app}=1-\cos(\mathbf f_i,\mathbf f_j)$ 由 ReID 特征余弦相似度刻画。相比局部贪心匹配，匈牙利算法能保证每帧多轨迹、多检测框之间的一对一全局分配，避免一个检测框被多条轨迹竞争。同时，系统对轨迹设置确认期、丢失缓存期和特征滑动平均：短暂遮挡不会立即删除轨迹，而长期未匹配轨迹才被终止，从而平衡身份稳定性与误检控制。

\section{低置信度目标、BIoU 与裁剪策略}
真实视频中，远距离行人或被遮挡行人常产生低置信、小尺寸或边界偏移的检测框。若直接丢弃这类检测框，会降低 IDR 和 MOTA；若直接将其初始化为新轨迹，又可能引入误检。因此本文采用低置信度补充匹配策略：高置信框用于常规匹配和新轨迹初始化，低置信目标不直接确认新身份，而只允许与已有轨迹进行补充匹配。该设计借鉴 ByteTrack 的思想：低分框中既包含噪声，也可能包含被遮挡目标的有效观测，关键是限制其只承担补回已有轨迹的作用。

IoU 严格比较预测框与检测框的交并比；BIoU（Buffered IoU）则在关联阶段临时扩大预测框或候选框的搜索区域，更适合处理检测框偏小、遮挡导致边界不准的情况。BoostTrack++（参考文献[9]）中的例子表明：即使检测框 $D_i$ 的原始置信度较低，只要它与长期未匹配轨迹 $T_j$ 的预测框仍有较高重叠，就可能是被遮挡目标的有效观测，而不应被简单丢弃。该扩展既不修改标注框，也不放宽评价指标，而是在匹配阶段提高弱目标召回，并继续通过门控阈值限制误匹配。外观分支中，普通框裁剪容易把背景、相邻行人或地面纹理一并输入 ReID 网络；姿态感知裁剪则尽量围绕人体主体区域截取，使特征更关注衣着、轮廓和局部结构，从而降低背景对身份相似度的干扰。

\section{双层实验数据与迭代设置}
本文采用双层实验设计。第一层使用 MOT17/MOT20[3,10] 的公开数据格式和评价协议，对齐标准 MOT 任务定义；第二层构建 Wuzhou\_MidRoad 真实场景验证集，用于观察方法在实际监控视角下的表现。该视频约 99 秒、2475 帧，包含密集通道、行人交叉和尺度变化。本文将人工标注与核验后的 \texttt{gt.txt} 整理为 MOT 格式，其中包含 2474 帧、117 个行人身份和 33846 个标注框，并与跟踪器输出逐帧匹配评估。需要说明的是，公开 MOT 协议层用于对齐任务定义和展示拥挤场景难度；Wuzhou\_MidRoad 则作为真实场景自标注验证层，两者不混作同一测试集。

实验设置为五个逐步增强的版本，并尽量保持同一输入序列、同一评测脚本和同一 IoU 匹配阈值。V0 只使用 IoU 关联，作为最小可运行基线；V1 加入 Kalman 运动预测，主要验证短时遮挡和检测抖动是否改善；V2 加入 ReID 外观分支，重点观察目标交叉后的身份恢复；V3 使用姿态感知裁剪，减少背景和相邻行人对 ReID 特征的污染；V4 引入 BIoU 与低置信度目标搜索扩展，目标是提高遮挡、小目标和边界偏移情况下的召回。该设计使每个版本对应一个明确问题：轨迹是否更连续、身份是否更稳定、低分框是否能补回已有目标。图\ref{fig:ablation} 中蓝线展示身份匹配质量随模块加入逐步提升，绿线将身份切换减少转换为身份稳定性上升趋势，说明系统是在相同评价口径下逐步定位并修正误差来源。

\section{ReID 训练过程与评价指标}
外观分支使用 OSNet-AIN / MobileNet Hybrid 作为 ReID 特征提取模型，在 Market1501（128,234 训练样本）与 MSMT17（32,058 验证样本）数据上训练。训练采用 SGD 优化器和 Cosine Annealing 学习率调度，batch size 为 32，输入尺寸为 $256\times128$。训练过程保留了 19 个 checkpoint（第 1、10、20、...、170、178 epoch）的关键统计量，包括训练损失（TrnLoss）、训练准确率（TrnAcc）、验证损失（ValLoss）、验证准确率（ValAcc）、验证 mAP@0.5 和学习率。本文保留离散记录点而不额外平滑，横向展示训练/验证损失对比、mAP 与验证准确率变化、学习率衰减阶梯和最佳 checkpoint（epoch 178，mAP@0.5 = 0.6272）。

评价指标采用 MOT 常用定义。IDP 表示身份匹配精确率，IDR 表示身份匹配召回率，IDF1 是二者调和平均；MOTA 综合考虑漏检、误检和身份切换；ID Switches 统计同一真实身份被分配到不同预测 ID 的次数。评价脚本在每帧内按 IoU 阈值和匈牙利算法匹配真实框与预测框，累计 TP、FP、FN 和身份切换。在本文任务中，IDF1 更侧重身份匹配质量，MOTA 更侧重检测与关联的整体稳定性，因此二者需要结合解释。

\section{实验结果与误差分析}
受控消融结果见表\ref{tab:demo}。基线版本已取得较高 IDF1，但仍存在身份切换；加入框裁剪与 ReID 后，身份匹配更稳定；姿态裁剪进一步减少背景干扰，使 ID Switches 降为 0。仅使用 Kalman 预测可以平滑轨迹，但缺少外观约束时仍可能在目标交叉场景中混淆身份，因此运动分支与外观分支需要联合使用。

\section{实现细节、可重复性与总结}
实现部分围绕可重复实验组织：评价模块统计 MOT 风格指标，跟踪模块完成 Kalman 状态预测、轨迹更新与外观特征平滑，裁剪模块提供标准框裁剪和姿态感知裁剪。受控消融和 Wuzhou 真实场景评估使用统一的评价口径，图表生成流程可由训练记录、视频抽帧和统计结果重新生成，避免手工截图与实验结果脱节。

从结果看，运动--外观双分支的价值在于把短时位置连续性和跨遮挡身份一致性合并到同一个关联框架中。Kalman 与匈牙利匹配解决多轨迹与多检测框的全局分配问题，ReID 与姿态裁剪缓解目标交叉后的身份混淆，BIoU 与低置信度扩展用于补回遮挡条件下的弱检测观测。真实场景评估进一步暴露出召回不足、远距离小目标和长时间遮挡仍是主要难点。本文的贡献更接近工程整合与场景适配，而非原创算法提出；因此重点放在模块动机、可重复实验路径、消融趋势和失败来源解释上。

综上，本文完成了从算法设计、训练过程、标准评测到自标注真实验证的完整 MOT 系统分析。当前不足主要有三点：真实场景各版本结果尚未完全量化，因而真实消融仍不如受控消融严格；ReID 评价侧重训练过程摘要，缺少 Rank-1/CMC 等检索指标；失败案例仍以统计和典型观察为主，尚未逐段标出长遮挡、密集交叉和小目标漏检片段。后续可围绕这些问题补充真实消融表、ReID 检索评价和更细粒度的错误案例分析。
